## Global Rules

### Color Palette (Cyber-Terminal IDE Theme)

| Role | Hex | CSS Variable |
|------|-----|--------------|
| Primary Background | `#0A0A0E` | `--color-bg-primary` |
| Secondary/Card | `#121218` | `--color-bg-secondary` |
| Glass Overlay | `rgba(255, 255, 255, 0.03)` | `--color-bg-glass` |
| Text Primary | `#EDEDF0` | `--color-text-primary` |
| Text Muted | `#888893` | `--color-text-muted` |
| Accent/Select | `#CCFF00` (Electric Lime) | `--color-accent-lime` |
| Accent/Data | `#00F0FF` (Laser Cyan) | `--color-accent-cyan` |
| Border | `rgba(255, 255, 255, 0.08)` | `--color-border-subtle` |

**Color Notes:** Aggressively dark slate backgrounds with extremely controlled, vivid neon accents. Completely avoid generic purple/blue gradients.

### Typography (Bespoke/Editorial)

- **Heading Font:** `Bricolage Grotesque` (from local fonts folder). Distinctive, slightly brutalist, editorial.
- **Data/UI Font:** `JetBrains Mono` (Google Fonts). Developer-grade, precise, highly structured for VINs/Metrics.
- **Mood:** Premium, highly-engineered, bespoke, professional tool.

### Motion & Effects

- **Data Reveals:** Use CSS `animation-delay` staggered cascades for lists and metrics.
- **Micro-interactions:** Glowing borders (`box-shadow: 0 0 15px rgba(204, 255, 0, 0.2)`) on active inputs/buttons rather than simple opacity drops.
- **Depth:** Radial ambient glows and extremely faint grid-patterns on empty backgrounds.

## Component Specs

### CVA Architecture (Class Variance Authority)

- All shared UI elements (Buttons, Inputs, Cards) MUST be built using generic `cva` patterns to enforce style consistency.

### Buttons

```css
/* Primary Cyber Button */
.btn-primary {
  background: var(--color-accent-lime);
  color: #000000;
  border-radius: 4px;
  font-family: 'JetBrains Mono', monospace;
  font-weight: 600;
  text-transform: uppercase;
  transition: all 150ms ease;
  box-shadow: 0 0 10px rgba(204, 255, 0, 0.2);
}

.btn-primary:hover {
  box-shadow: 0 0 20px rgba(204, 255, 0, 0.5);
  transform: translateY(-1px);
}
```

### Cards

```css
.card {
  background: var(--color-bg-secondary);
  border: 1px solid var(--color-border-subtle);
  border-radius: 8px; /* Sharper corners than typical SaaS 16px */
  backdrop-filter: blur(12px);
  transition: all 200ms ease;
}
```

## Anti-Patterns (Do NOT Use)

- ❌ **Space Grotesk, Inter, Roboto, Arial.**
- ❌ **Rounded, bubbly corners (e.g., `rounded-2xl`).** Keep corners sharper (`rounded-md`, `rounded-lg`) to reflect precision.
- ❌ **Generic purple/pink startup gradients.**
- ❌ **Flat gray backgrounds.** Always use true-black slates with subtle noise/grid textures.
- ❌ **Instant state changes.** Use strict 150ms-300ms transitions.
